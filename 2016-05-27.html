学んだこと

/*コメント*/ の意義

  • 素晴らしいプログラマが書いたコードでも、素人の自分が直接読むのは難しい
  • Class の先頭に「この Class の存在意義」がコメントの形で書かれていたりする
  • 先にコメントを読んでコンテキストを把握しておけば、コードが多少読みやすくなる
  • （という当たり前のことを、いくつかパッチを書かせていただいてから気付く……）

コーディングスタイルの大切さ

  • 変数の種類ごとに prefix が決まっている
  • グローバル: g / メンバ: m / 引数: a
  • 1k 行を超える C++ を泳ぐときに助かる
  • 道具として使える変数をエディタ上で検索できる（特にメンバ変数の m）

テストの書き方

  • 実装とテストは必ずセット
  • テストも負債化しないように、適宜リファクタリングをする
  • 自分の趣味コードでもテストを書くように
  • 「テストを書く」ことに対する恐怖心の薄れ
  • 「テストを書きたい」という気持ちの変化
